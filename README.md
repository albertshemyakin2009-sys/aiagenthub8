# AIAGENTHUB — Static Demo (with Requests, Support & Auth Modal)

Обновлённый прототип маркетплейса AI-агентов.

## Что есть

- Тёмный продуктовый UI, единый стиль.
- Главная (`index.html`) — hero, live-панель, избранные агенты.
- Каталог (`catalog.html`) — поиск, фильтры, раскрывающиеся карточки агентов с примерами.
- О нас (`about.html`) — миссия, для кого, качество, планы.
- Техподдержка (`support.html`) — только технические вопросы.
- Запросы (`requests.html`) — идеи по платформе, партнёрства, предложения агентов.
- Настройки (`settings.html`) — демо-профиль, выбор языка, валюты.
- Админка (`admin.html`) — демо-панель модерации агентов.
- `script.js`:
  - мок-данные агентов,
  - логика фильтров,
  - раскрытие "Подробнее",
  - разделение Support / Requests,
  - i18n RU/EN,
  - модальное окно регистрации/входа + "Продолжить как гость"
    (демо-режим, без реального backend),
  - клики "Подключить" требуют авторизации или гостевого режима.

## Как запускать

1. Открыть `index.html` напрямую в браузере
   или
2. Запустить любой статиκ-сервер в корне:

```bash
python3 -m http.server 4173
# или
npx serve .
```

## Как деплоить на Vercel / GitHub Pages

- Залить содержимое архива в репозиторий.
- Для Vercel:
  - Framework: `Other`
  - Build Command: —
  - Output Directory: `/`
- Для GitHub Pages:
  - Включить Pages из ветки `main` / `docs`.

## Дальнейшая интеграция

- Подключение реального backend-аутентификации:
  - реализовать /api/auth/register, /api/auth/login, /api/auth/me,
  - заменить алерты в `script.js` на реальные запросы.
- Для агентов:
  - хранить их описания в базе,
  - реализовать модерацию и статусы,
  - при необходимости — единый протокол вызова API агентов.
